# P10 剪辑台实现（阶段10探索与研究结论）

## 1. 文档目标

本文件用于回答两个问题：

1. 阶段10（片段剪辑与最终导出）现有改进计划是否完整。
2. 当前实现中是否存在功能或流程缺失，并给出可落地的补齐方案。

本文件面向后续研发落地，输出的是可执行方案，不是概念性建议。

---

## 2. 结论总览

结论：阶段10现有基础骨架，但存在**系统性链路断裂**，多个核心模块被 GitNexus 识别为零引用死代码，进入生产前必须先补充 P0 接入工作，再推进 P1～P3 能力扩展。

主要结论（已通过 GitNexus + 代码审计核实）：

1. `VideoEditorStage`、`createProjectFromPanels`、`migrateProjectData`、`VideoComposition` 四个核心组件均确认为死代码（`incoming: {}`），整条 P10 链路当前**对用户完全不可见**。
2. 前端路由三处断点：`WorkspaceStageContent` 无 editor 分支 / `VideoStageRoute` 未传 `onEnterEditor` / `useWorkspaceStageNavigation` 中 editor 项永久硬编码 `disabled: true`。
3. 后端两处缺陷：`/editor/render` 路由不存在 / `SaveEditorProjectRequest` 类型只有 `projectData`，`saveProject` 请求体未携带 `episodeId`，后端会返回参数错误。
4. 数据稳定性漏洞：GET API 读取 `projectData` 后**未调用** `migrateProjectData`/`validateProjectData`，旧版本数据直接透传会导致前端解析失败。
5. BGM 功能实为 UI 存根：`useEditorState` 中 `addBgm`/`removeBgm` 已实现但 `VideoEditorStage` 未暴露，时间轴 BGM 行只有标签无任何交互元素。
6. 素材库左侧面板全为 placeholder 占位文本，没有实际实现。
7. 阶段10计划里的 LUT/多平台导出/字幕样式设计器/片头片尾缺少接口契约、状态机与测试矩阵。

2. **编辑器核心类型**：片段、附属音频字幕、转场、BGM 轨道、渲染状态类型已定义（editor.types.ts）。
3. **预览层**：Remotion Player 与时间轴帧同步基础可用（RemotionPreview.tsx 实现完整）。
4. **时间轴层**：片段拖拽重排、播放头、缩放、片段选中、基础转场选择（4 种类型：none/fade/dissolve/wipe）可用。
5. **后端基础接口**：`/editor` 的 GET/PUT/DELETE 已存在，含 `requireProjectAuthLight` 权限校验。
6. **BGM 状态操作**：`useEditorState` 中 `addBgm`/`removeBgm`/`updateBgm` 已完整实现（useEditorState.ts:84+）；但**未暴露给 VideoEditorStage**。
7. **schema 迁移工具**：`migrateProjectData`/`validateProjectData` 在 `migration.ts` 中实现；但**未进入 API 读写路径**（GitNexus 确认 `incoming: {}`）。
8. **Remotion 合成层**：`VideoComposition` 的 BGM 渐入渐出、字幕轨道渲染逻辑已实现。
9. **I18n**：editor 命名空间多语言已全覆盖（messages/zh/ + messages/en/）。

### 4.3 数据稳定性风险

1. **GET 路径跳过迁移（B7 延伸）**：若用户有 v1.0 旧数据，前端以当前类型结构解析会导致类型断裂或渲染崩溃；`migration.ts` 的迁移工具已就绪但从未被调用（GitNexus 确认 `incoming: {}`）。
2. **`panel.duration` 单位歧义**：`createProjectFromPanels` 中用 `panel.duration * 30` 计算帧数（假设单位为**秒**）；而 `video.worker.ts` 的 `toDurationMs` 对 `> 1000` 的值视为毫秒。若 AI 将字段写为 `3000`（毫秒），入轨后片段时长将是预期的 1000 倍，整条时间轴错乱。

### 4.4 UI 存根（功能骨架存在，交互未实现）

1. **BGM 轨道**：`Timeline.tsx` BGM 行仅有标签占位，无新增/移除/裁剪/波形/音量操作元素；`addBgm` 钩子未接入界面。
2. **素材库左侧面板**：`VideoEditorStage.tsx:144-157` 全为 placeholder 文本（`t('editor.left.title')` + `t('editor.left.description')`），没有媒体列表、搜索、拖入时间轴等实现。
3. **字幕样式**：仅支持 `default`/`cinematic` 两种，目标为样式设计器（字体/描边/阴影/安全区）。
4. **导出反馈**：`handleExport` 仅调用 `alert()` 告知成功，无轮询进度展示。
5. **LUT 调色**：类型层、UI 参数层、Remotion 渲染层均未定义。

### 4.5 工程缺口

1. `/editor/render` 未在 `route-catalog.ts` 中登记（契约测试漏报）。
2. 缺少端到端回归脚本（保存→重载→导出→轮询→下载完整链路）。
3. 缺少性能预算定义（时长上限、素材数量上限、渲染 SLA）。
4. `createProjectFromPanels` 无幂等去重：快速多次进入剪辑台可能创建多条重复 `VideoEditorProject` 记录（待接入时需加唯一性校验）。

## 5. 对"现有阶段10计划"的补全建议

原计划项：前端入口、LUT、BGM、多平台导出、字幕设计、片头片尾。

以下是基于代码审计的**具体补充规范**：

### 5.1 B1-B3 前端路由补全（P0）

- `WorkspaceStageContent` 需增加 `'editor'` 分支，渲染 `<EditorStageRoute episodeId={episodeId} />`。
- `VideoStageRoute` 需从父级获取 `onEnterEditor` 回调并透传给 `VideoStage`。
- `useWorkspaceStageNavigation` 中 editor 项去掉硬编码 `disabled: true`，改为 `disabled: !stageArtifacts.hasVideo`。

### 5.2 B4-B5 saveProject 契约修复（P0）

- `useEditorActions.saveProject` 请求体改为 `{ episodeId, projectData: project }`（需从上下文或 props 获取 `episodeId`）。
- `SaveEditorProjectRequest` 类型补充 `episodeId: string` 字段（editor.types.ts:~125）。

### 5.3 B6 /editor/render 路由实现（P0）

- `POST /editor/render`：创建 BullMQ 任务，写入 `renderStatus = 'pending'`，返回 `{ taskId, status }`。
- `GET /editor/render?id=editorProjectId`：查询 `VideoEditorProject.renderStatus`，返回 `{ status, progress, outputUrl, error }`。
- 两条路由均需登记至 `route-catalog.ts`（契约测试覆盖）。
- 同一 `editorProjectId` 仅允许一个活跃渲染任务（提交前检查 `renderStatus === 'pending' | 'rendering'` 的去重）。

### 5.4 B7 数据加载路径接入 migration（P0）

- GET 路由返回前调用 `migrateProjectData(rawData)` → `validateProjectData(migrated)` 校验通过再响应。
- `createProjectFromPanels` 初始化后同样需经过 `validateProjectData`，确保入轨数据符合当前 schema。
- `panel.duration` 单位统一为**秒（浮点数）**，在 `createProjectFromPanels` 和 worker 两侧加注释；`toDurationMs` 移除 `> 1000` 毫秒判断分支，统一按"秒 × 1000"换算。

### 5.5 BGM 轨道交互接入（P1）

- `VideoEditorStage` 解构 `useEditorState` 时补充 `addBgm`/`removeBgm`/`updateBgm`（当前 lines 47-59 缺失）。
- `Timeline.tsx` BGM 行接入上述钩子，实现拖拽、缩放、音量滑块。

### 5.6 导出反馈 UI（P1）

- `handleExport` 改为提交任务后进入轮询模式（建议 2 秒间隔），展示进度条与状态文本。
- 提交前检测是否已有活跃渲染任务，避免重复提交。

### 5.7 计划层尚需纳入但当前文档未涵盖的模块

- 接口与数据契约规范（含 render 状态机字段与错误码）。
- 从阶段8/9产物自动入轨规则（lipSync URL 优先，降级使用 videoUrl）。
- 失败重试与幂等策略。
- 断点恢复（刷新后恢复编辑与导出状态）。
- 素材库左侧面板（P1）；字幕样式设计器（P2）；LUT 调色（P2）；片头/片尾模板化（P2）；多平台导出预设（P1）。

## 6. 阶段10目标流程（建议版）

### 6.1 E2E 主流程

1. 用户进入阶段10页面（projectId + episodeId）。
2. 系统加载 editor project：
   - 若存在则加载并迁移到最新 schema。
   - 若不存在则根据阶段8/9结果自动创建初始时间轴。
3. 用户完成编辑：片段顺序、转场、BGM、字幕样式、调色参数。
4. 用户保存项目（PUT /editor）。
5. 用户发起导出（POST /editor/render）。
6. 前端轮询导出状态（GET /editor/render?id=...）。
7. 成功后返回 outputUrl，支持下载/发布。
8. 失败时给出错误原因与重试入口。

### 6.2 导出状态机

pending -> rendering -> completed
pending -> rendering -> failed
failed -> rendering（重试）

约束：

1. 同一 editorProjectId 同时只能有一个活跃渲染任务。
2. 重试应保留最近一次失败信息与日志索引。

   - 响应：{ status, progress, outputUrl, error }

## 11. 风险与缓解

1. 风险：Remotion 导出时长长，任务超时。
   - 缓解：任务分级（draft/high）、超时重试、进度心跳。
2. 风险：素材 URL 失效导致导出失败。
   - 缓解：导出前预检素材可达性与时长元数据。
3. 风险：多平台参数差异导致成片兼容问题。
   - 缓解：平台预设白名单 + 导出前参数校验。
