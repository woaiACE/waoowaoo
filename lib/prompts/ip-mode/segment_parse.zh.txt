你是一位结构化文本解析专家。你的任务是将改写后的剧本文本解析为标准的结构化片段格式。

【改写后的剧本文本】
{rewritten_screenplay}

【解析要求】

将剧本文本拆分为独立的片段，每个片段包含以下信息：

```json
{
  "segments": [
    {
      "type": "dialogue|narration|action|transition",
      "speakerLabel": "说话者标签（对话类型时必填，其他类型为 null）",
      "text": "片段文本内容",
      "emotionTag": "neutral|happy|sad|angry|fearful|surprised|tender|excited|null",
      "stageDirection": "舞台指示或动作说明（可选）",
      "durationHint": 2.5
    }
  ]
}
```

【片段类型说明】
- **dialogue**: 角色对话，必须有 speakerLabel
- **narration**: 旁白或叙述文本
- **action**: 动作描写或场景描述
- **transition**: 转场指示（如：切到、淡出等）

【情绪标签说明】
- neutral: 平静/中性
- happy: 开心/愉快
- sad: 悲伤/伤感
- angry: 愤怒/生气
- fearful: 恐惧/害怕
- surprised: 惊讶/吃惊
- tender: 温柔/柔和
- excited: 兴奋/激动

【重要规则】
- 严格按 JSON 格式输出
- 每个对话片段必须有 speakerLabel
- durationHint 基于文本长度和语速估算（秒）
- 保持片段的原始顺序
- 不要合并或拆分语义完整的句子
