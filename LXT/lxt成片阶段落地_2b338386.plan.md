# LXT 成片阶段（基础版）落地计划

## 目标范围（本期）
- 新增 LXT `final-film` 阶段（位于“制作脚本”之后），一集内按“分镜=1行卡片”展示。
- 每行先落基础字段与操作位：文案、图片提示词、图片、视频尾帧图、视频提示词、视频、操作按钮组。
- 新增“资产库出厂角色/场景绑定”能力：在成片行中可绑定已有 LXT 资产（角色/场景），并写回行级结构。
- 保持与现有 LXT 资产库、任务系统、缓存刷新机制一致。

## 现状与复用结论
- LXT 已有完整阶段壳与保持挂载机制，可直接插入新阶段：[`src/app/[locale]/workspace/[projectId]/modes/lxt/LxtWorkspace.tsx`](src/app/[locale]/workspace/[projectId]/modes/lxt/LxtWorkspace.tsx)
- LXT 集级数据读取与刷新已集中封装，可直接扩展字段：[`src/app/[locale]/workspace/[projectId]/modes/lxt/hooks/useLxtWorkspaceEpisodeStageData.ts`](src/app/[locale]/workspace/[projectId]/modes/lxt/hooks/useLxtWorkspaceEpisodeStageData.ts)、[`src/app/[locale]/workspace/[projectId]/modes/lxt/LxtWorkspaceProvider.tsx`](src/app/[locale]/workspace/[projectId]/modes/lxt/LxtWorkspaceProvider.tsx)
- LXT 资产查询与资产操作已有成熟 hook：[`src/lib/query/hooks/useLxtAssets.ts`](src/lib/query/hooks/useLxtAssets.ts)
- 通用模式在“卡片化资产操作、任务状态覆盖、绑定写回”方面有可复用交互范式：[`src/app/[locale]/workspace/[projectId]/modes/novel-promotion/components/assets/CharacterCard.tsx`](src/app/[locale]/workspace/[projectId]/modes/novel-promotion/components/assets/CharacterCard.tsx)、[`src/lib/query/hooks/useTaskTargetStateMap.ts`](src/lib/query/hooks/useTaskTargetStateMap.ts)
- 后端已有 LXT episode PATCH/读取与资产 API，可沿用权限、校验、错误处理与缓存失效策略：[`src/app/api/lxt/[projectId]/episodes/[episodeId]/route.ts`](src/app/api/lxt/[projectId]/episodes/[episodeId]/route.ts)、[`src/app/api/lxt/[projectId]/assets/route.ts`](src/app/api/lxt/[projectId]/assets/route.ts)

## 目标数据契约（先定最小可用）
- 在 `LxtEpisode` 上新增 `finalFilmContent`，类型明确为 `String? @db.Text`（内部存储 JSON 字符串），与现有 `scriptContent` / `shotListContent` 风格保持一致。
- `finalFilmContent` 内部结构采用“行数组”模型（每行对应一个分镜）：
  - `shotIndex`
  - `copyText`
  - `imagePrompt`
  - `imageUrl`
  - `videoEndFrameUrl`
  - `videoPrompt`
  - `videoUrl`
  - `bindings`：`{ characterAssetIds: string[], sceneAssetId?: string }`
- 顶层建议加入 `version: 1` 与 `rows`，支持后续结构演进与容错解析。
- 行数据初始化来源：
  - 当前基础版优先由 `shotListContent` 解析出分镜行骨架（`shotIndex` + 默认空字段）。
  - `scriptContent` 暂不做强依赖解析，避免引入不稳定文本解析逻辑。

## 核心技术决策（补充）
- `finalFilmContent` 采用“行级字段更新”策略：前端提交 `shotIndex + patch`，后端做字段 merge 写回，避免整段覆盖造成并发冲突。
- 优先新增独立 API：[`src/app/api/lxt/[projectId]/final-film/route.ts`](src/app/api/lxt/[projectId]/final-film/route.ts)，并沿用 `requireProjectAuthLight + apiHandler`。
- 任务目标类型独立命名，避免与资产图任务混用：
  - `lxt_final_film_panel` 作为 `targetType`
  - `targetId` 采用 `${episodeId}:${shotIndex}` 复合键
- 成片行绑定仅保存资产引用关系，不修改资产实体；渲染时需要处理资产被删除后的降级显示。

## 实施步骤

### 阶段 A：阶段接线与基础页面骨架
- 在 LXT stage 枚举与 tab 中新增 `lxt-final-film`，接入阶段路由与保持挂载逻辑。
- 新增组件：`LxtFinalFilmStage`（基础布局：左侧集信息/筛选，右侧分镜行列表）。
- 增补 LXT i18n key（如 `tabs.finalFilm`），确保页签文案可本地化。
- 每行渲染固定槽位与占位按钮（先不做复杂生成链路）：
  - 文案编辑区
  - 图片提示词编辑区 + 图片展示位
  - 视频尾帧展示位
  - 视频提示词编辑区 + 视频展示位
  - 操作按钮组（预留：生成图片、设为尾帧、生成视频、重试等）

### 阶段 B：集级数据读写与行模型初始化
- 扩展 episode 读写路径，使 `finalFilmContent` 可读取；`episodes/[episodeId]/PATCH` 增加 `finalFilmContent` 白名单支持（兼容通路）。
- 新增 `final-film` 行级 PATCH 接口（推荐主通路），以字段级 merge 落库。
- 在 `useLxtWorkspaceEpisodeStageData` 增加 `finalFilmContent` 分支与初始化逻辑。
- 惰性初始化放在后端（读时 lazy upsert）：若无数据则基于 `shotListContent` 生成行骨架并落库，前端不直接整段初始化写入。

### 阶段 C：资产库出厂角色/场景绑定
- 在 `LxtFinalFilmStage` 接入 `useLxtAssets`，读取角色/场景（location）资产。
- 每行提供绑定入口（弹层/下拉均可，优先轻量弹层）：
  - 多选角色（character）
  - 单选场景（location）
- 绑定结果写入 `finalFilmContent.bindings`，并在行卡展示绑定摘要（头像/名称标签）。
- 绑定交互遵循现有 LXT 全局绑定语义：不修改资产实体，只记录“成片行到资产”的引用关系。
- 绑定渲染增加失效保护：若引用资产已删除，展示 fallback 标签并允许一键重绑。

### 阶段 D：按钮动作与任务框架预埋（不追求完整生成）
- 为行级按钮统一定义 action contract（前端事件 + 后端 API 占位）。
- 先打通最小闭环：
  - “生成图片”走独立 final-film 任务类型与 targetType，不复用资产图 targetType
  - “生成视频”接通独立提交链路（允许 handler 先返回未实现），但保留真实任务状态流转
- 接入任务状态覆盖（queued/processing/success/failed）展示骨架，复用任务查询 hook。
- 约束：任务状态展示以行级 target 为粒度，避免跨资产/跨分镜串状态。

### 阶段 E：验收与回归
- 回归 LXT 既有四阶段，确保新增阶段不影响原流程。
- 验证缓存刷新与切集切阶段的数据一致性。
- 验证绑定数据在刷新/重进后稳定恢复。

- M1：可见 `lxt-final-film` 页签 + 分镜行骨架渲染（无绑定）

- M3：角色/场景绑定可用并落库
- M3.5：`targetType` / `taskType` 注册完成，行级任务状态机可观测

## 风险与控制
- 数据结构风险：`finalFilmContent` 初版字段扩展频繁
  - 控制：用 versioned schema（`version: 1`）+ 容错解析
- 性能风险：分镜行较多导致渲染卡顿
  - 控制：先分页/折叠，必要时行虚拟化
- 语义冲突风险：成片绑定与资产全局绑定混淆
  - 控制：明确“成片行绑定=引用关系，不改资产主档”
- 并发覆盖风险：多入口同时更新同一集 `finalFilmContent` 导致字段丢失
  - 控制：行级 patch + 后端 merge 写入，避免整段覆盖
- 引用失效风险：资产被清空/删除后，`bindings` 指向孤儿 ID
  - 控制：渲染层降级展示 + 重绑入口 + 保存前过滤无效 ID
- 任务串态风险：复用资产 `targetType` 导致成片行状态混叠
  - 控制：独立 final-film `targetType` 与 `targetId` 规范
