## 二、分维度详细对比

### 2.1 流水线阶段架构

**2026 行业标准 7 阶段**（Seedance 2.0 / NovaKit / Genvid 公认）：
```
Script → Storyboard → Keyframe → Generate → Edit → Audio → Publish
```

**LXT 当前 5 阶段**：
```
资产库(lxt-assets) → 脚本生成(lxt-script/storyboard/final-script) → 成片(lxt-final-film)
```

**对齐分析**：

| 行业阶段 | LXT 对应 | 状态 | 说明 |
|----------|---------|------|------|
| Script | lxt-novel-to-script + lxt-script-to-storyboard | ✅ 对齐 | 小说→剧本→分镜 |
| Storyboard | shot_phase2a_image (4帧关键帧) | ✅ 对齐 | 4帧架构与行业 Keyframe 阶段一致 |
| Keyframe | lxt_final_film_image (4宫格→切割) | ✅ 对齐 | 生成后切割为独立帧 |
| Generate | lxt_final_film_video | ✅ 对齐 | 多模型支持 |
| Edit | **无** | ❌ 缺失 | 无 NLE 集成、无转场引擎 |
| Audio | **无**（voice-design 仅在资产阶段） | ❌ 缺失 | TTS/音效/BGM 未集成到成片 |
| Publish | **无** | ❌ 缺失 | 无序列化发布策略 |

**评级：★★★★☆ — 核心生成流程完整，后处理/发布层缺失。**

### 2.4 质量评估闭环

**这是 LXT 与 2026 行业标准之间最显著的代差**：

| 能力 | LXT | 行业标准 |

| 多智能体评审 | **无** | VQQA 3 专业 Agent（问题生成→回答→优化） |
| 自动化评分 | **无** | DramaBench 6 维评估 / PPEF 9 维评估 |
| 不合格修复循环 | **无** | Postwriter Plan→Draft→Validate→Repair→Revise |
| 变体筛选 | **无** — 每镜仅 1 个视频 | 纳米漫剧一次多选成功率 90%+ |
| 人工确认节点 | profileConfirmed（单点） | OmniAgent 超图多节点质量门控 |

**行业标准架构示例 — VISTA 多智能体系统**：
```
用户输入 → 结构化视频规划(时序场景拆解)
  → 生成候选池 → 成对锦标赛选择
  → 三维度多智能体评审(视觉/音频/语境)
  → 深度思考提示词代理(自省式推理→提示词修订)
  → 重新生成 → 迭代至达标
```

**评级：☆☆☆☆☆ — 完全缺失，是 P2-3 计划中的最高优先级项。**

### 2.5 角色一致性方案

**FilmWeaver 架构**（2026 行业标杆，Kling/Kuaishou 团队）是当前参照基准：

```
FilmWeaver 双级缓存：
  Shot Cache (长期): CLIP 检索前镜关键帧 → 维护角色/场景身份跨剪切
  Temporal Cache (短期): 压缩滑动窗口 → 确保连续流畅运动
  4 种工作模式：无缓存(开始)/仅时间/仅镜头/完整缓存

LXT 当前方案：
  1. videoSeed — 同集单种子锁定
  2. 前一镜尾帧注入(prevEndFrameSigned)
  3. 当前镜 4 帧全量注入
  4. 资产参考图注入(同一角色历史 imageUrl)
```

| 技术 | LXT | FilmWeaver/行业 | 差距 |
|------|-----|----------------|------|
| 种子锁定 | ✅ videoSeed | 行业推荐但脆弱 | 对齐 |
| 参考图注入 | ✅ 4 帧 + 尾帧 + 资产图 | Seedance 最多 9 图 | 对齐（数量略少） |
| CLIP 检索缓存 | ❌ | FilmWeaver Shot Cache | **代差** |
| 滑动窗口时序缓存 | ❌ | FilmWeaver Temporal Cache | **代差** |
| IP-Adapter/FaceID | ❌ | 行业标配（权重 0.7-0.85） | **代差** |
| LoRA 角色微调 | ❌ | 行业推荐（30min+/角色） | 差距 |
| DNA 锁定（40+ 属性） | ❌ | Artiroom 9/10 一致性 | **代差** |
| 角色护照多角度 | ❌ | 4 角度标准(正面/侧面/¾/全身) | 差距 |

**评级：★★★☆☆ — 基础技术（种子+参考图）已实施，但缺少 FilmWeaver 级双级缓存和 IP-Adapter/LoRA。**

## 四、核心结论

### 4.1 架构一致性判断：**基本一致但不完整**

LXT 的核心生成流程（Prompt 工程→分镜→关键帧→视频生成→模型适配）与 2026 行业标准高度一致。特别是 Prompt 工程维度（摄影机词汇、表演指导、4 帧架构）已达一流水平。

但 2026 行业已进入 **"生成+评估+优化"闭环时代**（以 VISTA/VQQA/DramaBench 为代表），而 LXT 仍停留在 **"生成即结束"的线性模式**。这是最核心的代差。

### 4.2 优势领域（保持领先）

1. **Prompt 工程**：摄影机词汇 16+16+20 种、表演指导 58 个术语 7 大类、4 帧 7 要素 + 递进变化 — 已达行业一流
2. **角色提取**：15+ 结构化字段、9 种关系类型、非人类专属字段、人称视角映射 — 超越行业平均
3. **模型适配**：Strategy 模式统一 5 供应商 — 与行业模型路由趋势一致
4. **数据模型**：行级 patch 合并、版本化 JSON、资产唯一键 — 设计前瞻

### 4.3 关键差距（按影响力排序）

| 排名 | 差距 | 行业影响 | LXT 计划 |
|------|------|---------|---------|
| 1 | **质量评估闭环** | VISTA 提升 60%，DramaBench 6 维 | P2-3（8-10 周后） |
| 2 | **FilmWeaver 级角色一致性** | 双级缓存 + IP-Adapter | 部分 P0（种子+参考图已做），核心未启动 |
| 3 | **视频后处理（VFI/转场/音频/调色）** | 行业标配 | P1-1 + P1-3 + P2-2 |
| 4 | **智能模型路由** | 降成本 60-70% | P1-2（基本多模型） |
| 5 | **多角度参考图体系** | 角色护照标准 | 未计划 |
| 6 | **英文 Prompt 补齐** | Imagen/OpenAI 质量 | P1-6 |

### 4.4 建议优先级调整

基于行业对标分析，建议将以下项从 P2 提升至 P1：

1. **P2-3（多智能体评审）→ P1**：这是 2026 年最显著的代差，DramaBench + 轻量 Critic Agent 可显著提升产出质量
2. **FilmWeaver 双级缓存 → P1**：CLIP 检索 + 滑动窗口是 2026 角色一致性的核心机制，ROI 极高
3. **变体生成+筛选 → P1**：一次多选是行业降本增效的关键手段
